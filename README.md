## What is Varinth?

Varinth is a **multi-agent, proof-first codebase verification platform**. It takes any claim — written by a developer, generated by an AI tool, or produced by an LLM — and cross-references it against your actual repository using a three-agent swarm of specialized AI judges.

Instead of asking _"Does this sound right?"_, Varinth asks _"Can this be proven from the code?"_

> **Varinth extracts atomic claims → retrieves semantic evidence → runs a concurrent verification swarm → outputs a fully auditable Proof Object with exact file paths and line numbers.**

| **Protocol** | MCP (Model Context Protocol) | Native Claude Desktop & Cursor integration |
| **Auth** | Supabase Auth (JWT) | Session management, route protection |
| **Rate Limiting** | In-memory token bucket (10 req/min) | Prevents API abuse and LLM cost overruns |

## Features

### 🤖 Three-Agent Verification Swarm
- **Critic** — Identifies logical gaps, naming mismatches, and missing context in claims
- **Verifier** — Evaluates each code snippet independently, flagging `supports_claim` or `contradicts_claim`
- **Judge** — Synthesizes all evidence into a final verdict, producing a **Grounded Correction** with file paths and line numbers whenever a claim is contradicted

### ⚡ High-Performance Execution
- **~3 second** end-to-end verification (down from 45+ seconds in V1)
- Concurrent swarm execution via `asyncio.gather`
- Warm repository cache avoids full `git clone` on subsequent audits
- SQLite embedding cache bypasses NVIDIA API for unchanged files

### 🔍 Question-Only Mode
- Submit a question with **no pre-written answer**
- Varinth retrieves codebase context, drafts a grounded response, and self-audits its own output
- Transforms from a verification tool into an **exploratory codebase intelligence engine**

### 📡 MCP Native Integration
- Run Varinth directly inside **Claude Desktop** or **Cursor Composer**
- `varinth_verify` — audit any AI-generated answer
- `varinth_compare` — compare two AI answers head-to-head with trust scores

### 🛡️ Enterprise Security
- **NVIDIA NemoClaw / OpenClaw**: Sandboxed filesystem access, network whitelisting via `openclaw-sandbox.yaml`
- **Colang Guardrails** (`nemoclaw/rails.co`): Prompt injection blocking, off-topic request rejection
- **Path Traversal Protection**: `os.path.realpath` canonicalization preventing directory escape
- **JWT Auth**: Every route protected via Supabase session verification

### 🔁 Pydantic Repair Loop
- If an LLM returns malformed JSON, Varinth intercepts the error, formats a correction prompt with the full validation trace, and requests a repaired payload — **without crashing the audit run**

## Quick Start

### Prerequisites
- Python 3.12+, Node.js 18+
- [Supabase](https://supabase.com) project (free tier works)
- [NVIDIA API Key](https://build.nvidia.com) (free tier available)

### 1. Backend Setup

```bash
cd backend
python -m venv venv
.\venv\Scripts\activate          # Windows
pip install -r requirements.txt

# Create .env
cp .env.example .env
# Fill in: SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, NVIDIA_API_KEY

# Run database migrations
# Execute schema.sql then phase2_schema.sql in your Supabase SQL editor

uvicorn app.main:app --reload
```

### 2. Frontend Setup

```bash
cd frontend
npm install
cp .env.local.example .env.local
# Fill in: NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, NEXT_PUBLIC_API_URL

npm run dev
```

### 3. Claude Desktop MCP Integration

Add to `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "varinth": {
      "command": "python",
      "args": ["-m", "app.mcp_server"],
      "cwd": "/path/to/varinth/backend",
      "env": {
        "SUPABASE_URL": "...",
        "SUPABASE_SERVICE_ROLE_KEY": "...",
        "NVIDIA_API_KEY": "nvapi-..."
      }
    }
  }
}
```

### 4. Docker Deployment

```bash
# Full stack deployment
docker-compose up --build
```

## NVIDIA NemoClaw Blueprint

Varinth ships with full [NVIDIA NemoClaw](https://github.com/NVIDIA/NemoClaw) blueprint support:

- **`nemoclaw/config.yml`** — Registers Varinth's swarm agents with NVIDIA AI endpoints
- **`nemoclaw/rails.co`** — Colang guardrails blocking prompt injection and off-topic requests
- **`openclaw-sandbox.yaml`** — OpenShell policies restricting filesystem access to code directories only
- **`openclaw.json`** — Agent-level model and instruction bindings

To enable private GPU inference, set in `.env`:
```
NEMOCLAW_ENABLED=True
NEMOCLAW_URL=http://localhost:9000/v1
```
